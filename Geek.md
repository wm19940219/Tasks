## 你觉得什么是Geek，什么又是 Geek 精神？

在我看来，Geek 最初指对技术、计算机、电子设备等领域有狂热兴趣的人，但今天它的含义已经泛化——Geek 是那些愿意深入探究事物原理、不满足于“能用就行”的人。
而 Geek 精神则体现为：有好奇心，不满足于表面，喜欢追问“它为什么这样工作”；有动手能力，愿意亲自尝试、搭建、调试，而不是只做旁观者；乐于分享与协作，相信开源的力量，乐于把自己的经验传递给他人。

## GitHub 是干什么的？ Pull Request、Fork 这些词你是怎么理解的，又是从哪里学到的？

GitHub 是一个基于 Git 的代码托管平台，但它远不止“存代码的地方”。它是全球最大的开源协作社区，人们在这里：分享项目，参与他人项目，构建自己的技术档案，用 Issues、Pull Requests、Projects 等功能进行团队协作。
Pull Request就是请求将自己的修改合并回原项目，它是开源协作的核心：我改完代码，通过 PR 告诉维护者“我改了这些，请考虑合并”；Fork则为复制别人的仓库到自己的账户下，即我要基于这个项目做自己的修改，但不会影响原项目。Pull Requests和Fork通常成对出现，是开源协作的标准工作流。
这些词我是从接受到这个任务开始，去摸索，询问AI，并再次进行理解。
