## 框架

我发现 **HTML** 写起来其实是有模板的  
标题（**Heading**）是通过`<h1> - <h6>` 标签来定义的。  
段落是通过标签 `<p>` 来定义的。  
链接是通过标签 `<a>` 来定义的。（提示:在 **href** 属性中指定链接的地址。）  
图像是通过标签 `<img>` 来定义的.（注意： 图像的名称和尺寸是以属性的形式提供的。）  
即房子已经全部弄好了，装修由你去完成，你可以在这里进行创作！


# HTML – 搭骨架，像拼乐高
核心思想：**HTML**就是网页的“毛坯房”，标签就是砖块。`<body>` 里面放用户看得见的一切，顺序很重要！

## 📦 基础框架（必背！）
```
html  

<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>网页标题</title>
</head>
<body>
    <!-- 所有用户能看到的内容都放在这里 -->
</body>
</html>
```

## 🧩 往 `<body>` 里加这些代码，会出现什么？

|代码	|效果	|备注|
| :----- | :------: | -----: |
|`<h1>标题</h1>`	|大标题（最大字号）	一个页面最好只有一个`h1`|
|`<p>文字</p>`	|普通段落	|会自动换行|
|`<a href="网址">链接文字</a>`	|可点击跳转	|加 **target="_blank"** 新标签页打开|
|`<img src="图片.png">`	|显示图片	|路径错了就破图|
|`<ul><li>项目1</li><li>项目2</li></ul>`	|无序列表	|导航栏常用|
|`<div class="xxx">`	|容器，无样式	|配合 **CSS** 用来布局|
|`<button>按钮</button>`	|可点击按钮	|配合 **JS**做交互|

## 💡 在这个招新网站里，我写了这些关键HTML：
固定导航栏：`<nav>` + `<ul>` + `<li><a href="#区块id">` → 点击跳转到页面内对应位置。

卡片区域：每个方向用一个 `<div class="card">`，里面包图片区和文字区。

问答区域：每个问题用 `<button class="faq-question">`，答案用相邻的 `<div class="faq-answer">`。

**Contact Us**：邮箱用 `<span>` 显示，复制按钮用 `<button id="copyEmailBtn">`。

🤪 一开始我把 `<img>` 的 `src` 写成了 `"我的猫.jpg"`，结果图片没出来  
💡 路径必须真实存在，空格和中文容易翻车，建议图片名全英文+下划线。
