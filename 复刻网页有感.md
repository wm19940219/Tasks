# 复刻！！！

## 一、HTML – 搭骨架
写 <!DOCTYPE html>，分 head（标题、CSS链接）和 body（内容）。

导航栏：<nav> + <ul> + <li> + <a>，href="#id" 页面内跳转，外部链接加 target="_blank"。

中央Logo：<section class="hero"> 包一张艺术字图片。

四个卡片：每个用 <div class="card">，内分图片区和文字区，文本从素材复制。

About Us + Contact Us：按新要求改成垂直布局，Contact Us 独立区块，包含邮箱、按钮、二维码、QQ-Group/E-mail/Github。

问答：每个问题用 <button class="faq-question">，答案用相邻 <div class="faq-answer">。

返回顶部按钮：<a href="#" class="go-top">🚀</a>。

## 二、CSS – 刷样式
全局：白色背景，深色文字，padding-top 留出导航栏空间。

导航栏：position: fixed + backdrop-filter: blur，链接 hover 变色。

中央区域：背景图 background_Homepage.png，艺术图居中，margin-top 负值微调位置。

卡片：display: flex，.card:nth-child(even) 反转顺序实现左右交替，加 border-radius、box-shadow、hover 上浮。

About Us：三个特性用 flex 水平卡片，右下角蝴蝶图片绝对定位，hover 显示小字。

Contact Us：居中卡片，邮箱复制区，社交链接，二维码。

问答：浅蓝背景（或背景图），.faq-answer 默认 display: none，.show 时显示。

响应式：@media 手机版改单列。

悬浮浮动：所有图片加 transition 和 :hover { transform: translateY(-5px); }。

## 三、JavaScript – 加交互
复制邮箱：navigator.clipboard.writeText，配合自定义确认框（确定/取消）。

问答折叠：点击问题，切换答案的 show 类，同时改变 +/− 图标。

QQ Group 点击：显示临时二维码浮层，关闭按钮或点击空白关闭。

返回顶部：平滑滚动 window.scrollTo({ top:0, behavior:'smooth' })。

导航栏锚点偏移：监听点击，计算目标位置减去导航栏高度。

## 四、调试与踩坑
图片不显示：文件名大小写、空格、路径错误 → 统一用小写+下划线，相对路径。

CSS 不生效：类名拼错，忘记加 px → F12 检查，看 computed。

JS 报错：元素 id 未定义，事件监听未等 DOM 加载 → 用 defer 或将 script 放 body 末尾。

部署后样式乱：图片路径未更新到远程，需 git add 并推送。

## 五、部署到 GitHub Pages
本地 git init，add .，commit -m "xxx"。

远程仓库 git remote add origin git@github.com:用户名/仓库名.git。

git push -u origin main。

仓库 Settings → Pages → Branch main，文件夹 /root → Save。

访问 https://用户名.github.io/仓库名/。

注意：主文件必须叫 index.html，根目录不能有冲突的 README 导致显示错误。

## 六、总结与感悟
HTML：语义化标签，合理使用 id/class。

CSS：Flex 布局真香，hover 动画提升体验，响应式不能忘。

JS：事件监听 + DOM 操作，学会用 classList.toggle 省去大量代码。

部署：Git 命令熟练后很快，Pages 自动构建，更新只需 git push。
